## 1. 项目概述和技术架构

### 1.1 项目背景

Heartopia 游戏网站是一个面向全球玩家的游戏资讯平台，需要支持多语言以服务不同地区的用户。项目采用 Next.js 15 + next-intl 实现了完整的国际化解决方案。

### 1.2 支持的语言

项目当前支持 **5 种语言**：

| 语言代码 | 语言名称 | URL示例 | 说明 |
|---------|---------|---------|------|
| `en` | 英语 (English) | `/guides/beginner` | 默认语言，无前缀 |
| `th` | 泰语 (ไทย) | `/th/guides/beginner` | 带语言前缀 |
| `ko` | 韩语 (한국어) | `/ko/guides/beginner` | 带语言前缀 |
| `ja` | 日语 (日本語) | `/ja/guides/beginner` | 带语言前缀 |
| `fr` | 法语 (Français) | `/fr/guides/beginner` | 带语言前缀 |

### 1.3 技术栈

**核心框架**:
- **Next.js 15** - React 全栈框架，使用 App Router
- **React 18** - UI 库
- **TypeScript** - 类型安全

**国际化方案**:
- **next-intl** - Next.js 专用的国际化库
- **gray-matter** - MDX 文件的 frontmatter 解析
- **marked** - Markdown 转 HTML

**内容管理**:
- **MDX** - Markdown + JSX，支持在文档中嵌入组件
- **文件系统路由** - 基于文件结构的内容组织

### 1.4 架构设计原则

**1. URL 策略：as-needed 前缀**
- 默认语言（英语）不带前缀，保持 URL 简洁
- 其他语言带语言代码前缀，便于识别和 SEO
- 示例：
  - 英语：`/guides/beginner`
  - 泰语：`/th/guides/beginner`

**2. 内容 Fallback 机制**
- 优先加载目标语言内容
- 如果目标语言内容不存在，自动回退到英语
- 确保所有语言版本都能正常访问

**3. SSG（静态站点生成）优先**
- 文章页在构建时预渲染所有语言版本
- 提升性能和 SEO 效果
- 减少服务器负载

**4. 命名空间隔离**
- 翻译文件按功能模块组织（common、home、article 等）
- 避免键名冲突
- 便于团队协作和维护

## 2. 核心配置文件详解

### 2.1 路由配置 (`src/i18n/routing.ts`)

这是多语言系统的核心配置文件，定义了所有语言相关的路由规则。

**文件位置**: `src/i18n/routing.ts`

```typescript
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  // 支持的语言列表
  locales: ['en', 'th', 'ko', 'ja', 'fr'],

  // 默认语言
  defaultLocale: 'en',

  // URL 前缀策略
  // 'as-needed': 默认语言(en)无前缀，其他语言带前缀
  localePrefix: 'as-needed',

  // 启用自动语言检测
  localeDetection: true
});
```

**配置项说明**:

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `locales` | `string[]` | 所有支持的语言代码列表 |
| `defaultLocale` | `string` | 默认语言，用于无前缀的 URL |
| `localePrefix` | `'always' \| 'as-needed' \| 'never'` | URL 前缀策略 |
| `localeDetection` | `boolean` | 是否启用自动语言检测 |

**localePrefix 策略对比**:

| 策略 | 默认语言 URL | 其他语言 URL | 适用场景 |
|------|-------------|-------------|---------|
| `'as-needed'` | `/guides` | `/th/guides` | 推荐，SEO 友好 |
| `'always'` | `/en/guides` | `/th/guides` | 所有语言统一格式 |
| `'never'` | `/guides` | `/guides` | 不推荐，无法区分语言 |

### 2.2 请求配置 (`src/i18n/request.ts`)

处理每个请求的语言加载逻辑，负责动态导入对应的翻译文件。

**文件位置**: `src/i18n/request.ts`

```typescript
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  // 从请求中获取语言代码
  let locale = await requestLocale;

  // 验证语言代码是否有效
  if (!locale || !routing.locales.includes(locale as 'en' | 'th' | 'ko' | 'ja' | 'fr')) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    // 动态导入对应的翻译文件
    messages: (await import(`../messages/${locale}.json`)).default
  };
});
```

**工作流程**:

1. **获取语言代码**: 从请求的 URL 中提取语言代码（如 `/th/guides` → `th`）
2. **验证有效性**: 检查语言代码是否在支持列表中
3. **回退处理**: 如果无效，使用默认语言 `en`
4. **加载翻译**: 动态导入对应的 JSON 翻译文件

**动态导入的优势**:
- 按需加载，减少初始包体积
- 每个语言的翻译文件独立，便于维护
- 支持代码分割，提升性能

### 2.3 导航组件 (`src/i18n/navigation.ts`)

创建支持多语言的导航组件，自动处理语言前缀。

**文件位置**: `src/i18n/navigation.ts`

```typescript
import { createNavigation } from 'next-intl/navigation';
import { routing } from './routing';

export const { Link, redirect, usePathname, useRouter } =
  createNavigation(routing);
```

**导出的组件和函数**:

| 导出项 | 类型 | 说明 | 使用场景 |
|-------|------|------|---------|
| `Link` | 组件 | 自动处理语言前缀的链接组件 | 替代 Next.js 的 `<Link>` |
| `redirect` | 函数 | 支持多语言的重定向函数 | 服务端重定向 |
| `usePathname` | Hook | 获取当前路径（不含语言前缀） | 客户端组件 |
| `useRouter` | Hook | 支持多语言的路由器 | 客户端导航 |

**使用示例**:

```typescript
import { Link } from '@/i18n/navigation';

// 自动添加语言前缀
<Link href="/guides/beginner">
  Beginner's Guide
</Link>

// 渲染结果：
// 英语: <a href="/guides/beginner">
// 泰语: <a href="/th/guides/beginner">
// 韩语: <a href="/ko/guides/beginner">
```

## 3. 首页多语言实现

### 3.1 组件结构

首页是一个客户端组件，使用 `'use client'` 指令实现交互功能。

**文件位置**: `src/app/[locale]/page.tsx`

**核心导入**:

```typescript
'use client';

import { useTranslations } from 'next-intl';
import { Link } from '@/i18n/navigation';
import Image from 'next/image';
```

**关键特点**:
- **客户端渲染**: 支持状态管理和交互功能（如代码复制、FAQ 展开等）
- **useTranslations Hook**: 客户端组件专用的翻译 Hook
- **自定义 Link 组件**: 自动处理语言前缀的链接

### 3.2 翻译使用方式

首页使用了多个翻译命名空间，按功能模块组织翻译内容。

**代码示例** (`src/app/[locale]/page.tsx:40-46`):

```typescript
export default function HomePage() {
  const t = useTranslations('home');        // 首页专用翻译
  const tc = useTranslations('common');     // 通用翻译（导航、按钮等）
  const tcl = useTranslations('coreLinks'); // 核心链接翻译
  const tf = useTranslations('faq');        // FAQ 翻译
  const tm = useTranslations('meta');       // SEO 元数据翻译
  const tt = useTranslations('tools');      // 工具翻译

  return (
    <div>
      <h1>{t('heroTitle')}</h1>
      <p>{t('heroSubtitle')}</p>
      <Link href="/codes">{tc('codes')}</Link>
    </div>
  );
}
```

**命名空间设计优势**:

| 命名空间 | 用途 | 示例键 |
|---------|------|-------|
| `home` | 首页专用内容 | `heroTitle`, `heroSubtitle` |
| `common` | 全站通用内容 | `home`, `guides`, `codes` |
| `coreLinks` | 核心链接卡片 | `link1Title`, `link1Desc` |
| `faq` | FAQ 问答 | `q1`, `a1`, `title` |
| `meta` | SEO 元数据 | `defaultTitle`, `defaultDescription` |
| `tools` | 工具相关 | `exportHeatmap`, `dpCalculator` |

**优势**:
- 避免键名冲突（不同模块可以有相同的键名）
- 便于团队协作（不同开发者负责不同命名空间）
- 易于维护（修改某个模块的翻译不影响其他模块）

## 4. 文章页多语言实现

### 4.1 路由设计

文章页使用动态路由 `[...slug]`，支持多层级路径结构。

**文件位置**: `src/app/[locale]/[...slug]/page.tsx`

**路由示例**:

```
URL: /guides/beginner
→ locale: 'en', slug: ['guides', 'beginner']

URL: /th/content/fishing-guide
→ locale: 'th', slug: ['content', 'fishing-guide']

URL: /ko/building/build-tutorial
→ locale: 'ko', slug: ['building', 'build-tutorial']
```

**路由结构解析**:
- `[locale]`: 动态语言参数（en, th, ko, ja, fr）
- `[...slug]`: 捕获所有路径段，支持多层级
- 第一个 slug 段是分类（category）
- 后续 slug 段是文章路径

### 4.2 SSG 配置

文章页采用静态站点生成（SSG）策略，在构建时预渲染所有页面。

**代码示例** (`src/app/[locale]/[...slug]/page.tsx:14-33`):

```typescript
// SSG 配置：构建时预渲染所有页面
// 不使用 ISR，不允许访问未预渲染的页面
export const dynamicParams = false;

export async function generateStaticParams() {
  // SSG: 预渲染所有文章详情页
  const allContentSlugs = await getAllContentPaths();
  const locales = ['en', 'th', 'ko', 'ja', 'fr'];

  return allContentSlugs.flatMap((slug) =>
    locales.map((locale) => ({ locale, slug }))
  );
}
```

**工作原理**:

1. **getAllContentPaths()**: 扫描 `src/content/` 目录，获取所有 MDX 文件路径
2. **flatMap 生成路径**: 为每个文章生成 5 种语言版本的路径
3. **构建时预渲染**: Next.js 在构建时为每个路径生成静态 HTML
4. **dynamicParams: false**: 禁止访问未预渲染的页面，返回 404

**生成的路径示例**:

```javascript
[
  { locale: 'en', slug: ['guides', 'beginner'] },
  { locale: 'th', slug: ['guides', 'beginner'] },
  { locale: 'ko', slug: ['guides', 'beginner'] },
  { locale: 'ja', slug: ['guides', 'beginner'] },
  { locale: 'fr', slug: ['guides', 'beginner'] },
  // ... 更多文章
]
```

**SSG 的优势**:
- **性能**: 静态 HTML，无需服务端渲染
- **SEO**: 搜索引擎可以直接抓取完整内容
- **成本**: 减少服务器负载，降低运营成本
- **可靠性**: 即使服务器故障，静态页面仍可访问

### 4.3 服务端翻译

文章页是服务端组件，使用 `getTranslations` 函数获取翻译。

**代码示例** (`src/app/[locale]/[...slug]/page.tsx:72-75`):

```typescript
export default async function ContentPage({ params }: PageProps) {
  const { locale, slug } = await params;

  // 服务端获取翻译
  const t = await getTranslations('article');
  const tCommon = await getTranslations('common');

  return (
    <div>
      <h1>{frontmatter.title}</h1>
      <p>{t('lastUpdated')}: {frontmatter.date}</p>
      <Link href="/">{t('breadcrumbHome')}</Link>
    </div>
  );
}
```

**客户端 vs 服务端翻译对比**:

| 特性 | 客户端 (`useTranslations`) | 服务端 (`getTranslations`) |
|------|---------------------------|---------------------------|
| 使用场景 | `'use client'` 组件 | 服务端组件（默认） |
| 调用方式 | Hook | async 函数 |
| 性能 | 需要加载 JS | 无需 JS，更快 |
| SEO | 需要等待渲染 | 直接输出 HTML |
| 交互性 | 支持状态和事件 | 仅静态内容 |

**选择建议**:
- **首页**: 使用客户端翻译（需要交互功能）
- **文章页**: 使用服务端翻译（SEO 优先，内容为主）
- **导航栏**: 可以使用服务端翻译（静态内容）

### 4.4 内容加载与 Fallback

使用 `getLocalizedContent` 函数加载 MDX 内容，支持自动回退。

**代码示例** (`src/app/[locale]/[...slug]/page.tsx:77-88`):

```typescript
// 获取分类（slug 的第一部分）
const category = slug[0];           // 'guides'
const contentSlug = slug.slice(1);  // ['beginner']

// 使用工具函数读取内容（支持 fallback）
const result = await getLocalizedContent(category, locale, contentSlug);

if (!result) {
  notFound();  // 如果内容不存在，返回 404
}

const { frontmatter, content } = result;
```

**内容路径解析**:

| URL | locale | category | contentSlug | 文件路径 |
|-----|--------|----------|-------------|---------|
| `/guides/beginner` | `en` | `guides` | `['beginner']` | `src/content/en/guides/beginner.mdx` |
| `/th/content/fishing-guide` | `th` | `content` | `['fishing-guide']` | `src/content/th/content/fishing-guide.mdx` |

**Fallback 流程**（详见第5部分）:
1. 尝试加载目标语言内容
2. 如果失败且不是英语，回退到英语版本
3. 如果英语版本也不存在，返回 404

## 5. 内容管理和 Fallback 机制

### 5.1 内容目录结构

MDX 内容按语言和分类组织，支持多层级目录。

**目录结构**:

```
src/content/
├── en/                    # 英语内容（基准语言）
│   ├── guides/
│   │   ├── beginner.mdx
│   │   └── advanced.mdx
│   ├── codes/
│   │   └── active-verified.mdx
│   └── content/
│       ├── fishing-guide.mdx
│       └── cooking-system.mdx
├── th/                    # 泰语内容
│   ├── guides/
│   │   └── beginner.mdx   # 已翻译
│   └── codes/             # 其他文件回退到英语
├── ko/                    # 韩语内容
├── ja/                    # 日语内容
└── fr/                    # 法语内容
```

**内容组织原则**:
1. **英语为基准**: 所有内容必须有英语版本
2. **按需翻译**: 其他语言可以选择性翻译
3. **自动回退**: 未翻译的内容自动显示英语版本
4. **保持结构一致**: 所有语言的目录结构应保持一致

### 5.2 Fallback 机制实现

`getLocalizedContent` 函数实现了智能的内容回退机制。

**文件位置**: `src/lib/content.ts:55-98`

**核心代码**:

```typescript
export async function getLocalizedContent(
  category: string,
  locale: string,
  slug: string[]
): Promise<ContentData | null> {
  const fileName = slug[slug.length - 1];
  const subPath = slug.slice(0, -1);

  // 1. 尝试读取指定语言版本
  let filePath = path.join(
    process.cwd(),
    'src/content',
    locale,           // 如: 'th'
    category,         // 如: 'guides'
    ...subPath,
    `${fileName}.mdx`
  );

  try {
    const fileContent = await fs.readFile(filePath, 'utf8');
    const { data, content } = matter(fileContent);
    return { frontmatter: data as Frontmatter, content };
  } catch {
    // 2. 如果失败且不是英语，回退到英语版本
    if (locale !== 'en') {
      filePath = path.join(
        process.cwd(),
        'src/content',
        'en',
        category,
        ...subPath,
        `${fileName}.mdx`
      );

      try {
        const fileContent = await fs.readFile(filePath, 'utf8');
        const { data, content } = matter(fileContent);
        return { frontmatter: data as Frontmatter, content };
      } catch {
        return null;
      }
    }
    return null;
  }
}
```

**Fallback 流程图**:

```
用户访问 /th/guides/beginner
    ↓
尝试读取 src/content/th/guides/beginner.mdx
    ↓
    ├─ 成功 → 返回泰语内容
    └─ 失败 → 尝试读取 src/content/en/guides/beginner.mdx
              ↓
              ├─ 成功 → 返回英语内容（Fallback）
              └─ 失败 → 返回 null（404）
```

**实际应用场景**:

| 场景 | 泰语文件 | 英语文件 | 返回结果 |
|------|---------|---------|---------|
| 场景1 | ✅ 存在 | ✅ 存在 | 返回泰语内容 |
| 场景2 | ❌ 不存在 | ✅ 存在 | 返回英语内容（Fallback） |
| 场景3 | ❌ 不存在 | ❌ 不存在 | 返回 404 |

**优势**:
- **渐进式翻译**: 可以逐步翻译内容，不影响网站运行
- **用户体验**: 即使内容未翻译，用户也能看到英语版本
- **维护成本**: 不需要为每种语言维护完整的内容库

## 6. SEO 和元数据处理

### 6.1 动态元数据生成

文章页使用 `generateMetadata` 函数生成多语言 SEO 元数据。

**文件位置**: `src/app/[locale]/[...slug]/page.tsx:35-70`

**核心代码**:

```typescript
export async function generateMetadata({ params }: PageProps): Promise<Metadata> {
  const { locale, slug } = await params;
  const baseUrl = 'https://www.heartopiagame.com';

  // 英语使用无前缀路径，其他语言带前缀
  const fullPath = locale === 'en'
    ? `/${slug.join('/')}`
    : `/${locale}/${slug.join('/')}`;

  // 获取内容以提取标题和描述
  const category = slug[0];
  const contentSlug = slug.slice(1);
  const result = await getLocalizedContent(category, locale, contentSlug);

  const title = result?.frontmatter?.title || slug[slug.length - 1];
  const description = result?.frontmatter?.description || '';

  return {
    title,
    description,
    alternates: {
      canonical: fullPath,
      languages: {
        'en': `/${slug.join('/')}`,
        'th': `/th/${slug.join('/')}`,
        'ko': `/ko/${slug.join('/')}`,
        'ja': `/ja/${slug.join('/')}`,
        'fr': `/fr/${slug.join('/')}`,
        'x-default': `/${slug.join('/')}`
      }
    },
    openGraph: {
      title,
      description,
      url: `${baseUrl}${fullPath}`,
      type: 'article'
    }
  };
}
```

**关键 SEO 元素**:

1. **canonical**: 规范链接，避免重复内容问题
2. **languages**: hreflang 标签，告诉搜索引擎不同语言版本
3. **x-default**: 默认语言版本（英语）
4. **openGraph**: 社交媒体分享元数据

**解决方案**:

3. 如果需要，创建对应语言的内容文件
